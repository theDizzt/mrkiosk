# [앱 개발자용] 키오스크 상태별 ArUco ID 출력 가이드 (최종본)

키오스크 화면의 상태가 변할 때마다 아래 규칙에 따라 **0 ~ 1023 사이의 정수 ID**를 산출하여 화면 우측 하단에 해당 번호의 마커 이미지를 렌더링해야 합니다.

---

### [시스템 공통 참조] 고유 메뉴 ID (0 ~ 29) 매핑표
*(※ 주의: 이 항목은 화면에 아루코 마커로 띄우는 값이 아닙니다. 외부 컨트롤러의 목표 입력 기준이자, 하단 Phase 01 및 Phase 10의 비트 계산식에 대입하기 위해 앱 내부 데이터베이스가 반드시 지켜야 하는 공통 인덱스 규격입니다.)*

* **Category 1 (Coffee):** 메뉴 ID `0 ~ 5`
* **Category 2 (Tea):** 메뉴 ID `6 ~ 11`
* **Category 3 (Ade/Juice):** 메뉴 ID `12 ~ 17`
* **Category 4 (Beverage):** 메뉴 ID `18 ~ 23`
* **Category 5 (Blended):** 메뉴 ID `24 ~ 29`

---

### 1. Phase 00: 초기 화면 및 카테고리 탭 (ID: 0 ~ 160)
화면이 대기 상태이거나 메뉴를 탐색하는 기본 화면입니다.

* **초기 매장/포장 선택 화면:** **`ID = 0` (고정 출력)**
* **카테고리 탭 (음료 리스트) 화면:** * 계산식: `ID = Category 번호(1~5) * 32`
  * Coffee 탭 화면 (Cat. 1) ➔ **ID: 32**
  * Tea 탭 화면 (Cat. 2) ➔ **ID: 64**
  * Ade/Juice 탭 화면 (Cat. 3) ➔ **ID: 96**
  * Beverage 탭 화면 (Cat. 4) ➔ **ID: 128**
  * Blended 탭 화면 (Cat. 5) ➔ **ID: 160**

---

### 2. Phase 01: 메뉴 상세 옵션 선택 (ID: 256 ~ 447)
메뉴를 터치하여 '옵션 팝업창'이 열렸을 때, 사용자의 선택 상태를 실시간으로 갱신하여 마커를 출력합니다.

* **계산식:** `ID = 256 + (MenuHash * 32) + (Temperature * 8) + (Sugar * 4) + (Ice * 2)`
* **변수 정의:**
  * **`MenuHash` (메뉴 해시값):** 각 카테고리 내에서 해당 메뉴가 몇 번째 위치하는지를 나타냅니다. `(공통 참조 ID % 6)`으로 계산되며 아래 값을 따릅니다.
    * **Hash 0:** 카테고리의 1번째 메뉴 (참조 ID 0, 6, 12, 18, 24)
    * **Hash 1:** 카테고리의 2번째 메뉴 (참조 ID 1, 7, 13, 19, 25)
    * **Hash 2:** 카테고리의 3번째 메뉴 (참조 ID 2, 8, 14, 20, 26)
    * **Hash 3:** 카테고리의 4번째 메뉴 (참조 ID 3, 9, 15, 21, 27)
    * **Hash 4:** 카테고리의 5번째 메뉴 (참조 ID 4, 10, 16, 22, 28)
    * **Hash 5:** 카테고리의 6번째 메뉴 (참조 ID 5, 11, 17, 23, 29)
  * `Temperature`: 미선택(`0`), ICED(`1`), HOT(`2`)
  * `Sugar`: 미선택(`0`), 선택 완료(`1`)
  * `Ice`: 미선택(`0`), 선택 완료(`1`)
* *(산출 예시: 14번 메뉴(Ade 카테고리의 3번째 ➔ Hash 2)의 옵션창에서 ICED(1)를 선택하고 당도/얼음은 아직 안 골랐다면 ➔ `256 + (2 * 32) + (1 * 8) + 0 + 0` = **ID 328** 출력)*

---

### 3. Phase 10: 장바구니 검증 (ID: 512 ~ 628)
장바구니 화면에 진입했을 때, 담겨 있는 메뉴의 고유 ID를 전송합니다.

* **계산식:** `ID = 512 + (장바구니에 담긴 메뉴의 공통 참조 ID * 4)`
* *(산출 예시: 14번 메뉴가 장바구니에 담긴 화면 ➔ `512 + (14 * 4)` = **ID 568** 출력)*

---

### 4. Phase 11: 최종 결제 모듈 안내 (ID: 768 고정)
결제 수단(신용카드, 간편결제 등)을 묻는 화면부터 카드를 삽입하는 전체 결제 단계입니다.

* **결제 화면 전체:** **`ID = 768` (고정 출력)**

---

### 5. 주문 완료 및 루프 리셋 (매우 중요)
결제가 성공하여 **초기 대기 화면(매장/포장 선택)**으로 돌아왔을 때, 즉시 화면 우측 하단에 **`ID = 0`** 마커를 다시 출력해야 합니다. 
*(AR 가이드 엔진은 이 `768(결제) -> 0(초기화)`의 상태 변화를 감지하여 주문 성공 이펙트를 렌더링하고 내부 메모리를 초기화합니다.)*
